### Key Findings

- Legacy field: `Coin.RarityRating` (string, DB `rarity_rating`)
- Primary modern storage: `CoinReference` table with catalog validation via `CatalogRegistry`
- Existing referenceText/referenceUrl are external-link fallbacks, not primary RIC storage
- Current database state: 0 coins, 0 coin references (development seed)

### Proposed Migration Strategy

One-time, idempotent backfill after `AutoMigrate` and `seedCatalogRegistry`, guarded by `app_settings` marker `LegacyRarityRatingReferenceBackfillV1`.

**Key Rules:**

- Parse and normalize catalog names (RIC, SEAR, SPINK, etc.)
- Respect volume requirements (RIC/RPC/SNG require volume; SEAR/CRAWFORD/etc. do not)
- Skip ambiguous values (e.g., bare "RIC 207" without volume) instead of inventing structure
- Validate through `CoinReferenceService.NormalizeAndValidateOne`
- Use `certainty:"legacy-import"` for all backfilled references
- Skip without failing startup; log reasons for every skipped value
- Preserve `rarity_rating`, `reference_text`, `reference_url` columns (non-destructive)
